## 二、OCR 容错分析

| 问题 | OCR错误 | 检索结果 | 实际表现 |
|------|---------|----------|----------|
| 健的技术条件 | 健→键 | 4条，已容错 | 回答568字，准确引用技术要求 |
| 健的抗拉温度 | 健→键，温度→强度 | 3条 | 未匹配"抗拉强度"，返回"未找到" |
| 龟键的要求 | 龟→? | 5条 | 检索到内容但 LLM 判断文档无"龟键"，正确拒答 |

**结论：**
- 轻度 OCR 错误（单个同音字 + 正确上下文词）可以被容错，如"健的技术条件"因"技术条件"提供了足够语义信号
- 中度 OCR 错误（两个关键词都错）容错失败，如"抗拉温度"与"抗拉强度"语义差异过大
- BGE Embedding 有一定中文容错能力，但依赖关键词层面的语义匹配

## 三、自检模块准确性

### 3.1 自检逻辑分析

当前自检基于**字符级重叠率**（答案用字 ÷ 检索文本用字的交集比例）：

```
overlap = len(answer_chars ∩ retrieval_chars) / len(answer_chars)
has_evidence = overlap > 0.3 OR avg_score > 0.4
possible_hallucination = overlap < 0.2 AND avg_score < 0.3
should_refuse = NOT has_evidence AND avg_score < 0.2
```

### 3.2 发现的问题

| 问题 | 现象 | 根因 |
|------|------|------|
| 拒答时仍显示"有据" | "文档中未找到相关信息" 11个字几乎都在检索文本中出现 | 字符级重叠不考虑语义，共用汉字多 |
| Python编程规范→有据 | 5条检索结果相关度0.3-0.5 | score_threshold=0.3 过低，噪声过多 |
| 590 MPa→无结果 | 检索0条 | 纯数值查询语义匹配失败，需关键词倒排兜底 |
| 拒答检测需 score<0.2 | 几乎不会触发 | 噪声检索分数也在0.3-0.5 |
| confidence 计算 | 拒答时有0.36置信度 | 重叠率×2与检索分取min，拒答不应有正置信度 |

### 3.3 自检数据

```
正常问答:  evidence=True,  confidence=0.73
无关问题:  evidence=True,  confidence=0.36  ← 应判定为无依据
OCR容错:   evidence=True,  confidence=0.90
拒答(有检索): evidence=True, confidence=0.73  ← 应判定为拒答
拒答(无检索): evidence=False, should_refuse=True, confidence=0.0 ✓
```

## 六、优化记录 (2026-05-30)

### 6.1 自检模块 → LLM 二次判断

**改动**：`src/self_checker.py` 完全重写

- 从字符级重叠率 → LLM 判断（复用 Qwen3-coder:30b）
- 快速路径：拒答短语 + 短答案 → 直接返回 `should_refuse=True`
- 降级路径：LLM 调用失败时回退到改进的规则判断
- 修复了 `str.format()` 因文档含 `{}` 导致的 KeyError

**效果**：

| 场景 | 优化前 | 优化后 |
|------|--------|--------|
| 正常问答 | has_evidence=True (重叠率72%) | has_evidence=True, conf=0.95, reason="答案引用了文档中..." |
| 拒答(有检索) | has_evidence=True ✗ | should_refuse=True, conf=0.9 (快速路径) |
| 完全无关 | has_evidence=True ✗ | should_refuse=True, conf=0.9 |
| 检索无结果 | should_refuse=True ✓ | should_refuse=True, conf=0.1 |

### 6.2 检索阈值

**改动**：`config.yaml` score_threshold: 0.3 → **0.38**

- 过滤噪声但保留边界查询（如"第5条" 0.389）
- "Python编程规范" (0.375) 被正确过滤

### 6.3 分块策略

**改动**：`src/chunker.py` 重构

- `max_chunk_size`: 500 → 300, `overlap`: 50 → 80
- 增加逐级切分：双换行 → 单换行 → 硬切
- 章节标题强制分块（`3 技术要求`、`4.1 基本规则` 等）
- 每块前添加 `[第X页]` 页码前缀
- 分块数：5 → 13（12正文 + 1表格）

### 6.4 Prompt 优化

**改动**：`src/generator.py` PROMPT_TEMPLATE

- 强调"仔细检查每个片段，只要有一个相关就回答"
- Context 按页码排序，帮助 LLM 形成文档结构认知

### 6.6 BM25 混合检索

**改动**：`src/retriever.py` 增加 BM25 关键词索引 + 条件混合检索

- 使用 jieba 分词构建轻量 BM25 索引
- 条件启用策略：`max_emb ≥ 0.45` → 纯 embedding；`< 0.45` → 0.7×emb + 0.3×bm25
- BM25 仅兜底低分查询（纯数值、短条款），避免干扰正常语义检索

**关键效果**：`590 MPa` 从 0 条 → 5 条，BM25 成功救援纯数值查询
