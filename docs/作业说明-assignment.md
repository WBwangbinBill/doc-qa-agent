## 一、作业背景

我们是一家自研大模型产品公司，正在构建面向复杂业务场景的 Agent 框架。真实项目中，客户常提供扫描版 PDF、合同、产品手册、标准文件、金融材料、合规文档等非结构化资料。系统不仅要"能跑"，还要能解释、能测试、能定位问题、能适配不同业务。

本次作业以附件中的扫描版 PDF 为输入，模拟"文档理解 + 检索问答 + 可靠性验证"的真实交付场景。

**附件**：`GBT 1568-2008 键 技术条件.pdf` — 扫描件，通常没有可直接复制的文本层，包含中文正文、编号条款和表格内容。

> 请不要手工录入全文来绕过解析过程。

---

## 二、作业目标

请设计并实现一个最小可运行的"智能文档问答 Agent"原型。系统应能围绕附件 PDF 完成以下闭环：

1. 判断 PDF 类型并选择解析策略
2. 提取或识别 PDF 中的正文、条款编号、表格信息
3. 构建可检索知识库
4. 接收用户问题，检索相关证据
5. 生成答案，并返回来源页码/片段
6. 对答案做基本自检：是否有依据、是否可能幻觉、是否需要拒答
7. 给出测试方法和不同业务场景下的保障方案
