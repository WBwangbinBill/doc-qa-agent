# 作业说明

> Agent 开发工程师（大模型方向）技术笔试作业

---

## 一、题目背景

构建一个**智能文档问答 Agent**，能够对扫描版 PDF 文档进行理解、检索和问答。

**测试文档**：`GBT 1568-2008 键 技术条件.pdf` — 一份 4 页的机械行业国家标准，扫描件，无文本层。

---

## 二、核心要求

### 2.1 功能要求

| # | 要求 | 说明 |
|---|------|------|
| 1 | 扫描 PDF 解析 | 对扫描件进行 OCR 识别，提取正文和表格 |
| 2 | 文档结构化 | 保留条款编号、页码、表格结构 |
| 3 | 智能检索 | 基于语义的文档片段检索，支持模糊和容错查询 |
| 4 | 问答生成 | 基于检索结果生成准确答案，引用来源页码 |
| 5 | 答案自检 | 判断答案是否有依据、可能幻觉、需要拒答 |
| 6 | 测试覆盖 | 覆盖正文检索、表格检索、条款编号、无答案、边界等场景 |

### 2.2 技术约束

- 模型可选用 OpenAI / DeepSeek / 本地部署
- 向量库可选用 ChromaDB / FAISS / Milvus / 内存检索
- 不能手工录入全文 — 必须走 OCR + 检索管道
- 不能硬编码答案 — 必须由 LLM 根据检索结果生成
- API Key 不能写进代码 — 通过环境变量或配置文件管理

### 2.3 提交流

- 代码仓库（GitHub）
- 5 篇设计文档
- 10 个以上测试用例
- AI 工具使用记录
-（可选）演示视频

## 三、测试用例清单

根据题目要求的 5 类问题场景设计：

| # | 问题 | 类型 | 预期行为 |
|---|------|------|---------|
| 1 | "键的材料要求是什么" | 正文检索 | 有答案，引用第3页，提到抗拉强度 ≥ 590 MPa |
| 2 | "键的表面处理要求" | 正文检索 | 有答案，引用相关页码 |
| 3 | "键的尺寸公差是多少" | 表格检索 | 有答案，引用表格所在页码 |
| 4 | "各类键的长度范围" | 表格检索 | 有答案，引用表格数据 |
| 5 | "第5条规定了什么" | 条款编号 | 有答案，准确引用第5条内容 |
| 6 | "这份标准里有没有提到钛合金键" | 无答案 | 拒答或明确说"文档中未找到" |
| 7 | "健的技术条件" | OCR容错 | 能容错检索到"键的技术条件" |
| 8 | "验收" | 模糊短问题 | 返回相关段落，不崩溃 |
| 9 | 空字符串 | 边界 | 不崩溃，返回提示 |
| 10 | 500字超长问题 | 边界 | 不崩溃，正常处理 |

## 五、设计原则

> 来自"洛水方法论"——先自问、再拆解、后执行

### 5.1 自问三问

1. **值不值得做？** — 扫描PDF问答是真实业务需求（合同审查、合规检查、技术文档查询），有明确应用场景
2. **能不做的有哪些？** — Web UI、多文档支持、流式输出、生产部署皆不在原型范围
3. **怎么验证？** — 5类10+测试用例，覆盖正确路径和异常路径

### 5.2 关键取舍

| 做了 | 没做（及原因） |
|------|---------------|
| PDF扫描件OCR解析 | Web UI — 原型用CLI足够 |
| 表格检测与提取 | 多文档 — 题目只要一份PDF |
| 条款编号保留 | 流式输出 — 不影响核心验证 |
| 分块+向量检索 | 生产级部署 — 原型目标 |
| LLM问答+来源引用 | |
| 答案自检（依据/幻觉/拒答） | |
| 10+测试用例+评估指标 | |
| AI使用记录+校验说明 | |

### 5.3 明确不做

- **手工录入全文** — 违反题目要求，面试会追问
- **硬编码答案** — 违反题目要求，面试会追问
