# 测试计划

## 一、测试用例

### 正文检索类（2 个）

| # | 问题 | 预期 |
|---|------|------|
| 1 | "键的材料要求是什么" | 有答案，引用第 2-3 页，提到 45 钢或同等材料 |
| 2 | "键的表面处理要求" | 有答案，引用相关页码 |

### 表格检索类（2 个）

| # | 问题 | 预期 |
|---|------|------|
| 3 | "键的尺寸公差是多少" | 有答案，引用表格所在页码 |
| 4 | "各类键的长度范围" | 有答案，引用表格数据 |

### 条款编号类（1 个）

| # | 问题 | 预期 |
|---|------|------|
| 5 | "第5条规定了什么" | 有答案，准确引用第 5 条内容 |

### 无答案类（1 个）

| # | 问题 | 预期 |
|---|------|------|
| 6 | "这份标准里有没有提到钛合金键" | 拒答或明确说"文档中未找到相关信息" |

### 模糊/容错类（2 个）

| # | 问题 | 预期 |
|---|------|------|
| 7 | "健的技术条件"（OCR 错误模拟） | 能容错检索到"键的技术条件" |
| 8 | "验收"（模糊短问题） | 返回相关段落，不崩溃 |

### 边界测试（2 个）

| # | 问题 | 预期 |
|---|------|------|
| 9 | 空字符串 | 不崩溃，返回"请输入问题" |
| 10 | 500 字超长问题 | 不崩溃，正常处理 |

---

## 二、评估指标

| 指标 | 计算方式 | 目标 |
|------|---------|:--:|
| 检索命中率 | 答案引用了相关段落的比例 | ≥ 80% |
| 来源准确率 | 引用页码正确的比例 | ≥ 70% |
| 拒答正确率 | 无答案时正确拒答的比例 | 100% |
| 表格识别率 | 表格内容被正确提取的比例 | ≥ 60% |
| 崩溃率 | 异常退出的请求比例 | 0% |

## 三、测试脚本设计

```python
# tests/test_qa.py

TEST_CASES = [
    {
        "id": 1,
        "question": "键的材料要求是什么",
        "type": "text",
        "expect_answer": True,
        "expect_page_range": [1, 5],
    },
    {
        "id": 3,
        "question": "键的尺寸公差是多少",
        "type": "table",
        "expect_answer": True,
        "expect_table_source": True,
    },
    {
        "id": 6,
        "question": "这份标准里有没有提到钛合金键",
        "type": "no_answer",
        "expect_refuse": True,
    },
    # ...
]

def run_all_tests(agent):
    results = []
    for tc in TEST_CASES:
        answer = agent.ask(tc["question"])
        passed, reason = evaluate(answer, tc)
        results.append({**tc, "passed": passed, "reason": reason, "answer": answer})
    return results
```

## 四、回归测试策略

- 每次修改检索/生成/自检代码后跑全量 10 个用例
- 记录每次测试结果，对比历史
- 新增功能 → 同步新增对应测试用例
