# 演示视频录制指南

> 预计时长：6-8 分钟

---

## 录制前准备

- [ ] 终端窗口最大化，字体调大（让观众看清）
- [ ] 确认 BGE 模型已缓存（`ls ~/.cache/huggingface/hub/models--BAAI--bge-large-zh-v1.5`）
- [ ] 确认环境变量已设置：`export DEEPSEEK_API_KEY="sk-xxx"`
- [ ] 确认 `cd /Users/wangbin/doc-qa-agent`

---

## 分段脚本

### 片段一：项目结构概览（30 秒）

**操作**：终端输入 `ls -R src/`

**口播**：
> "大家好，这是一个基于扫描 PDF 的智能文档问答 Agent 原型。项目采用模块化设计，包含 8 个核心模块：PDF 解析、表格提取、文本分块、向量化、检索、生成、自检，以及完整的测试套件。所有设计文档在 docs 目录下。"

---

### 片段二：安装依赖（30 秒）

**操作**：终端输入 `cat requirements.txt`

**口播**：
> "核心依赖很精简：PyMuPDF 处理 PDF，sentence-transformers 做向量化，openai SDK 调用 DeepSeek API，tesseract 做 OCR。不需要数据库服务，不需要 Docker。"

---

### 片段三：构建知识库（1 分钟）

**操作**：终端输入 `python main.py --parse-only`

**口播**：
> "运行 main.py --parse-only 开始构建知识库。系统首先判断 PDF 类型——这份 GB/T 1568-2008 是扫描件，4 页都没有文本层。接着逐页进行 tesseract OCR 识别，中文识别效果可以看到是稳定的。然后加载 BGE-large-zh-v1.5 模型，将文本分块后生成 1024 维向量，完成索引构建。整个过程不到 30 秒。"

---

### 片段四：问答测试 1 — 正文检索（40 秒）

**操作**：终端输入


## 录制注意事项

1. **终端用浅色背景**：视频更容易看清文字
2. **字体 16px 以上**：手机观看也能看清
3. **每个命令前停顿 1 秒**：方便后期剪辑
4. **口播说慢一点**：6-8 分钟刚好，太快了观众跟不上
5. **不要完美主义**：出现小卡顿、打错字重来，是真实开发过程的体现
